# 友達旅行用 Web アプリ 要件定義書

## 1. 概要

毎年友達と行く旅行の情報をまとめて確認できる、スマホ対応の静的 Web アプリを作成する。

旅行中・旅行後に必要となる以下の情報を、URL を知っている友達が簡単に閲覧できるようにする。

- 旅行の概要
- だいたいのスケジュール
- ホテルの詳細
- 必要経費の内訳
- 割り勘計算
- 麻雀の最終計算表

GitHub Pages で公開し、来年以降も旅行ページを追加していける構成にする。

---

## 2. 基本方針

### 2.1 利用者

- 主な閲覧者は、自分と友達
- URL を知っている人は閲覧可能
- ログイン機能は不要
- 画面上での編集機能は不要
- 旅行データの編集は開発者本人のみが行う

### 2.2 公開範囲

- GitHub Pages による公開を前提とする
- URL を知っている人が閲覧できればよい
- 完全な非公開性は求めない
- ただし、予約番号、個人情報、詳細すぎる住所、電話番号などは掲載しない方針とする

### 2.3 対応デバイス

- スマホでの閲覧を最優先する
- PC でも閲覧できるようにする
- モバイルファーストのレスポンシブ UI とする

---

## 3. 技術方針

### 3.1 推奨技術スタック

以下の構成を推奨する。

- Vite
- React
- TypeScript
- React Router
- GitHub Pages

### 3.2 データ管理

GitHub Pages の静的サイトとして運用するため、DB は使わない。

旅行データは TypeScript ファイル、または JSON ファイルとして管理する。

推奨構成例：

```txt
src/
  data/
    trips/
      2026-kagawa.ts
      2027-hokkaido.ts
    trips.ts
```

旅行ごとのデータは個別ファイルに分ける。

## 5. 経費・割り勘機能

### 5.1 目的

旅行後にまとめて入力した経費データをもとに、参加者間の最終精算額を自動計算する。

### 5.2 前提

- 旅行中のリアルタイム入力は不要
- 旅行後にまとめてデータを入力する
- 画面上での編集機能は不要
- 経費データはコードまたは JSON に記述する

### 5.3 機能要件

- 経費一覧を表示する
- 経費ごとに以下を管理する
  - 項目名
  - 金額
  - 支払者
  - 対象者
  - カテゴリ
  - メモ
- 対象者が一部メンバーのみの費用にも対応する
- 各メンバーの支払済額を計算する
- 各メンバーの本来負担額を計算する
- 差額をもとに最終的な精算方法を表示する
- 「誰が誰にいくら払えばよいか」を表示する

### 5.4 経費データ例

```ts
{
  title: "レンタカー代",
  amount: 30000,
  paidBy: "Aさん",
  participants: ["Aさん", "Bさん", "Cさん", "Dさん", "Eさん"],
  category: "transport",
  memo: "3日間分"
}
```

### 5.5 表示例

```txt
経費一覧

レンタカー代 30,000円
立替：Aさん
対象：Aさん、Bさん、Cさん、Dさん、Eさん
1人あたり：6,000円
```

```txt
最終精算

Bさん → Aさん：4,500円
Cさん → Dさん：2,000円
Eさん → Aさん：1,200円
```

### 5.6 端数処理

初期実装では 1 円単位で計算する。

ただし、将来的に以下を選べるよう拡張しやすい設計にする。

- 1 円単位
- 10 円単位
- 100 円単位

## 6. 麻雀精算機能

### 6.1 目的

旅行中に行った麻雀の点数を入力し、ウマ・オカを考慮した最終精算額を表示する。

### 6.2 前提

- 半荘ごとの結果を入力できるようにする
- 各半荘の素点をもとに精算額を計算する
- デフォルトルールは「1・2の点5」とする
- ウマ・オカを設定できるようにする
- 最終的に各参加者の合計収支を表示する

### 6.3 デフォルトルール

- レート：点5
- ウマ：1・2
  - 1位：+20
  - 2位：+10
  - 3位：-10
  - 4位：-20
- オカ：あり
- 返し点：30,000点
- 原点：25,000点

### 6.4 計算方針

各プレイヤーの半荘ごとの収支は、以下をもとに計算する。

```txt
素点ベースのスコア = (最終持ち点 - 返し点) / 1000
順位点 = ウマ
トップ賞 = オカ
最終スコア = 素点ベースのスコア + 順位点 + トップ賞
精算金額 = 最終スコア × レート係数
```

点5の場合、スコア 1.0 あたり 50 円として計算する。

### 6.5 入力項目

#### 半荘ごと

- 半荘名または番号
- プレイヤー名
- 最終持ち点
- 順位

順位は点数から自動判定できるようにする。
ただし同点時の扱いがあるため、必要に応じて順位を手動指定できる設計にしてもよい。

### 6.6 表示項目

- 半荘ごとの結果表
- 各プレイヤーの半荘ごとのスコア
- 各プレイヤーの合計スコア
- 各プレイヤーの最終収支
- 最終的な支払い一覧

### 6.7 データ例

```ts
{
  title: "2026 香川旅行 麻雀",
  rule: {
    rate: 0.5,
    uma: [20, 10, -10, -20],
    oka: 20,
    startPoint: 25000,
    returnPoint: 30000
  },
  games: [
    {
      label: "1半荘目",
      results: [
        { name: "Aさん", point: 42500 },
        { name: "Bさん", point: 28000 },
        { name: "Cさん", point: 21000 },
        { name: "Dさん", point: 8500 }
      ]
    }
  ]
}
```

### 6.8 表示例

```txt
麻雀最終結果

Aさん +2,400円
Bさん +300円
Cさん -800円
Dさん -1,900円
```

```txt
支払い例

Dさん → Aさん：1,900円
Cさん → Aさん：500円
Cさん → Bさん：300円
```

```ts

  date: string;

  time?: string;

  location?: string;
  memo?: string;
  url?: string;
```

### 7.3 Hotel

```ts

  name: string;
  stayDates: string[];

  checkOut?: string;
  address?: string;

  officialUrl?: string;
  bookingName?: string;

  name: string;
  members: string[];
};
```

### 7.4 Expense

```ts
export type Expense = {
  title: string;
  amount: number;
  paidBy: string;
  participants: string[];
  category?: "transport" | "hotel" | "food" | "activity" | "other";

```

### 7.5 MahjongData

```ts
export type MahjongData = {
  title: string;
  rule: MahjongRule;
  games: MahjongGame[];

  rate: number;
  uma: [number, number, number, number];
  oka: number;
  startPoint: number;
  returnPoint: number;

  label: string;
  results: MahjongPlayerResult[];
};

export type MahjongPlayerResult = {
  name: string;
  point: number;
  rank?: number;
};
```

## 10. 計算ロジック要件

### 10.1 割り勘計算

#### 入力

- 参加者一覧
- 経費一覧

#### 出力

- 各参加者の支払済額
- 各参加者の負担額
- 各参加者の差額
- 最終的な支払い指示

#### 計算手順

1. 各メンバーの支払済額を集計する
2. 各経費について、対象者数で割って負担額を計算する
3. 各メンバーごとに「支払済額 - 負担額」を計算する
4. プラスの人は受け取る側、マイナスの人は支払う側とする
5. 支払う側から受け取る側へ、差額が相殺されるように支払い一覧を生成する

---

### 10.2 麻雀計算

#### 入力

- 麻雀ルール
- 半荘ごとの各プレイヤーの点数

#### 出力

- 半荘ごとのスコア
- 合計スコア
- 円換算した収支
- 最終的な支払い指示

#### デフォルト設定

```ts
const defaultMahjongRule = {
  rate: 0.5,
  uma: [20, 10, -10, -20],
  oka: 20,
  startPoint: 25000,
  returnPoint: 30000,
};
```
